# Contributing to EliteAuth

Thanks for helping improve EliteAuth.

## Before opening a pull request

1. Open or reference an issue for substantial changes.
2. Keep each pull request focused on one problem.
3. Do not commit secrets, credentials, generated dependencies, or user data.
4. Preserve backwards compatibility unless the change is clearly documented.
5. Update documentation and examples when behavior changes.

## Local checks

```bash
npm install
npm run check
```

Test the website locally with:

```bash
npm run dev
```

## Commit messages

Use clear, imperative messages such as `Fix session validation expiry check`.
