# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Email **security@eliteauth.lol** with:

- A clear description of the issue
- The affected component or endpoint
- Reproduction steps or a minimal proof of concept
- The likely impact
- Logs or screenshots with secrets and personal data removed

Do not include active passwords, service-role keys, signing private keys, server secrets, or session tokens.

## Supported versions

Security fixes are applied to the current `main` branch. Tagged releases may receive fixes when explicitly marked as supported.
